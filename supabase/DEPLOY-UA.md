# Як задеплоїти проєкт — інструкція без комп'ютера (тільки браузер)

Ця інструкція для того, хто має лише браузер: жодних програм ставити не треба.
GitHub сам завантажить код на Supabase. Ви робите все мишкою на двох сайтах —
**Supabase** і **GitHub**.

Загальна ідея: ви даєте GitHub три «пароля» (secrets), кладете їх у захищений
сейф GitHub, а далі натискаєте одну кнопку — і код їде на сервер сам.

---

## Крок 1. Створити проєкт на Supabase

1. Зайдіть на https://supabase.com і зареєструйтесь (можна через GitHub-акаунт).
2. Натисніть **New project**, дайте назву, придумайте пароль до бази (запишіть
   його про всяк випадок), виберіть регіон, натисніть **Create new project**.
3. Зачекайте ~1–2 хвилини, поки проєкт створиться.

### Де взяти `project-ref`

Подивіться на адресу в браузері, коли ви всередині проєкту:

```
https://supabase.com/dashboard/project/abcdefghijklmnop
                                        ^^^^^^^^^^^^^^^^
                                        оце і є project-ref
```

Скопіюйте цей код — знадобиться далі (це буде `SUPABASE_PROJECT_ID`).

---

## Крок 2. Взяти токен доступу Supabase

Це «тимчасовий ключ», яким GitHub заходитиме у ваш Supabase.

1. Відкрийте https://supabase.com/dashboard/account/tokens
2. Натисніть **Generate new token**, дайте назву (напр. `github-deploy`).
3. Скопіюйте показаний токен **одразу** — його показують лише раз.
   Це буде `SUPABASE_ACCESS_TOKEN`.

---

## Крок 3. Взяти НОВИЙ ключ Anthropic

1. Відкрийте https://console.anthropic.com → розділ **API Keys**.
2. Натисніть **Create Key**, скопіюйте ключ. Це буде `ANTHROPIC_API_KEY`.
3. Якщо якийсь старий ключ колись світився (у чаті, листах) — видаліть його
   тут же кнопкою видалення. Старий ключ вважається небезпечним.

---

## Крок 4. Покласти всі три пароля у «сейф» GitHub

**Важливо: пароли вводяться тут, у сейфі GitHub — НЕ в чат Клоду.**

1. Відкрийте репозиторій на GitHub у браузері.
2. Зверху натисніть вкладку **Settings** (Налаштування).
3. Ліворуч у меню: **Secrets and variables** → **Actions**.
4. Натисніть зелену кнопку **New repository secret** і додайте по черзі три штуки
   (щоразу: вписати ім'я, вставити значення, **Add secret**):

   | Name (ім'я — точно так)  | Secret (значення)                     |
   | ------------------------ | ------------------------------------- |
   | `SUPABASE_ACCESS_TOKEN`  | токен із Кроку 2                      |
   | `SUPABASE_PROJECT_ID`    | project-ref із Кроку 1               |
   | `ANTHROPIC_API_KEY`      | новий ключ із Кроку 3                |

   Імена мають збігатися **буква в букву** (великими літерами, з підкресленнями).

---

## Крок 5. Натиснути кнопку запуску

1. У репозиторії відкрийте вкладку **Actions** (угорі).
2. Ліворуч виберіть **Deploy Supabase Functions**.
3. Праворуч натисніть **Run workflow** → ще раз **Run workflow** (зелена кнопка).
4. Зачекайте 1–3 хвилини. Зелена галочка = все завантажилось на сервер.
   Червоний хрестик = щось не так (найчастіше — помилка в імені або значенні
   одного з трьох секретів; перевірте Крок 4).

Готово — ваші функції тепер працюють на серверах Supabase.

---

## Крок 6. Підключити застосунок до сервера

У логах запуску (Крок 5) наприкінці буде рядок виду:

```
EXPO_PUBLIC_API_URL=https://<ваш-project-ref>.supabase.co/functions/v1
```

Це адреса вашого сервера. Її треба дати застосунку як налаштування
`EXPO_PUBLIC_API_URL`, щоб він звертався на сервер, а не на локальний дефолт.

---

## Крок 7 (необовʼязково). Реальні AI-детектори

Екран Result показує оцінки справжніх детекторів, якщо ви маєте їхні ключі.
Без них усе працює — просто застосунок рахує «приблизну» оцінку своєю евристикою,
і деплой **не падає**. Додавати ці ключі не обовʼязково.

Якщо ключі є — покладіть їх у той самий «сейф» GitHub (Крок 4), як окремі
repository secrets. Додавайте лише ті, що маєте:

| Name (ім'я — точно так)     | Що це                                     |
| --------------------------- | ----------------------------------------- |
| `GPTZERO_API_KEY`           | ключ GPTZero (головний детектор)          |
| `ORIGINALITY_API_KEY`       | ключ Originality.ai                       |
| `COPYLEAKS_API_KEY`         | ключ Copyleaks (разом з email нижче)      |
| `COPYLEAKS_EMAIL`           | email акаунта Copyleaks                   |

Потім знову натисніть **Run workflow** (Крок 5) — нові ключі поїдуть на сервер.
(Turnitin не додаємо: у нього немає публічного API, застосунок показує «N/A».)

---

## Важливо про безпеку (крок 13, ще попереду)

Зараз функції відкриті для всіх, хто знає адресу (входу/пароля немає — так
задумано на цьому етапі). Тому **не публікуйте адресу широко**: сторонні люди
зможуть витрачати ваш платний ключ Anthropic. Перед публічним запуском треба
зробити крок 13 з `PLAN.md` — захист входом (auth) і обмеження частоти запитів.
